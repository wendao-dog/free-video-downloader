## 一、确定产品的通用默认 TDK
### 1.1 标题（Title）规范
**结构模板**

建议采用三段式结构：页面名称 - 产品名称 | 产品核心介绍

**分隔符选择**

经过 SEO 实践验证，推荐使用以下分隔符组合：

+ 短横线（-）：用于连接相关词汇，如页面名称与产品名称之间
+ 竖线（|）：用于分隔不同层级的信息，如产品名称与核心介绍之间
+ 避免使用：下划线（_）、斜杠（/）、逗号（,）等不利于 SEO 的符号

这种组合方式有助于搜索引擎更好地识别标题的层次结构，提升关键词权重分配的准确性。

**长度控制**

+ 最佳长度：50-60 个字符（包括空格）
+ 最大长度：不超过 70 个字符，避免在搜索结果中被截断
+ 百度特殊性：百度会显示约 30 个汉字，建议核心关键词前置

**内容要求**

+ 必须包含页面主关键词，建议出现在标题前半部分
+ **每个页面标题必须独一无二，避免重复**
+ 语句通顺自然，符合用户搜索习惯
+ 避免关键词堆砌，保持可读性



### 1.2 描述（Description）规范
**功能定位**

描述是网页在搜索结果中的"广告文案"，需要在有限字数内吸引用户点击，同时覆盖核心关键词。



**长度控制**

+ **Google 最佳长度**：150-160 个字符（包括空格）
+ **百度最佳长度**：约 200 个字符
+ **建议策略**：前 150 字符放置最核心信息，后续内容作为补充



**内容要求**

+ 自然融入多个核心关键词及相关长尾词
+ 突出产品核心价值和差异化优势
+ 包含行动号召（如"立即使用"、"免费体验"等）
+ **每个页面描述必须独特，不可复制粘贴**
+ 语句流畅，避免单纯罗列关键词



**覆盖策略**

应尽可能覆盖以下维度的关键词：

+ 产品核心功能关键词
+ 目标用户群体关键词
+ 使用场景关键词
+ 行业/领域关键词
+ 品牌词和竞品对比词



### 1.3 关键词（Keywords）规范
**数量控制**

+ 最大数量：不超过 10 个，避免分散权重
+ 优先级排序：按重要性从高到低排列

**长度控制**

+ 单个关键词：2-8 个汉字为宜
+ 长尾关键词：不超过 15 个汉字
+ 避免使用过长的短语

**选择原则**

+ 优先选择搜索量大、竞争度适中的词
+ 结合页面实际内容，确保相关性
+ 包含 1-2 个品牌词或产品名称
+ 适当加入长尾关键词，提升精准流量

搜索引擎兼容性说明

+ Google：已基本不使用 Keywords meta 标签进行排名
+ 百度：仍会参考 Keywords 标签，建议保留
+ 其他国内搜索引擎（360、搜狗等）：仍有一定参考价值
+ 建议：虽然权重降低，但设置正确的关键词不会有负面影响，建议保留



## 二、为每个页面模板提供定制的 TDK
### 2.1 页面模板分类
页面模板是指具有相同结构和功能的一类页面。常见的页面模板包括：

+ **首页**：网站门户，展示核心价值
+ **列表页/分类页**：内容聚合页面（如文章列表、产品分类）
+ **详情页**：具体内容页面（如文章详情、产品详情）
+ **功能页**：特定功能页面（如搜索页、个人中心）
+ **静态页**：关于我们、帮助中心等



### 2.2 TDK 定制原则
**首页 TDK**

+ 标题：包含品牌词和核心业务关键词
+ 描述：全面介绍产品定位、核心功能、目标用户
+ 关键词：选择最核心的 10 个业务关键词

**列表页/分类页 TDK**

+ 标题：分类名称 + 品牌词，体现内容范围
+ 描述：说明该分类下的内容特点和价值
+ 关键词：该分类相关的精准关键词

**详情页 TDK**

+ 标题：内容标题 + 分类 + 品牌词
+ 描述：内容摘要或核心观点提炼（建议自动提取前 150 字并优化）
+ 关键词：从内容中提取的核心主题词（可通过算法自动提取）

**功能页 TDK**

+ 标题：功能名称 + 功能价值 + 品牌词
+ 描述：功能介绍和使用场景
+ 关键词：功能相关的操作词和需求词



### 2.3 TDK 优化管理表格
建议创建一份 TDK 规范表格，包含以下字段：

**表格必需字段**

+ 页面模板名称
+ 页面路径规则
+ 标题模板（含变量占位符）
+ 描述模板（含变量占位符）
+ 关键词模板（含变量占位符）
+ 动态变量说明（如文章标题、分类名称等）
+ 示例展示
+ 负责人
+ 更新时间

**使用场景**

+ 新增页面类型时，开发同学可查表确定 TDK 规则
+ 产品/运营同学可定期审核和优化 TDK 模板
+ 便于批量检查和优化现有页面的 TDK 质量



## 三、必需的 SEO Meta 标签清单
### 3.1 基础 Meta 标签（必需）
**标题标签**

+ Title 标签：网页最重要的 SEO 元素，必须独一无二

**描述和关键词标签**

+ Description meta 标签：影响点击率的核心因素
+ Keywords meta 标签：虽然权重降低，但建议保留（尤其针对百度）



### 3.2 字符编码与视口设置（必需）
**字符编码**

+ 使用 UTF-8 编码，确保中文正确显示

**移动端适配**

+ Viewport meta 标签：确保移动端正常显示，对移动搜索排名有影响



### 3.3 Open Graph 标签（强烈推荐）
**用途说明**

Open Graph 标签主要用于社交媒体分享（如微信、Facebook、Twitter 等），当用户分享链接时，会展示设置的标题、描述和图片。

**必需的 OG 标签**

+ og:title - 分享标题（可与页面 Title 一致或略有差异）
+ og:description - 分享描述
+ og:image - 分享图片（建议尺寸：1200x630 像素）
+ og:type - 页面类型（网站首页用 website，文章用 article）
+ og:url - 页面完整 URL（建议使用规范链接）
+ og:locale - 语言地区（如 zh_CN）
+ og:site_name - 网站名称

**图片要求**

+ 格式：JPG 或 PNG
+ 尺寸：推荐 1200x630px（比例 1.91:1）
+ 大小：不超过 5MB
+ 必须使用绝对路径（完整 URL）



### 3.4 Twitter Card 标签（可选，面向国际化）
如果产品面向海外用户，建议添加：

+ twitter:card - 卡片类型（如 summary_large_image）
+ twitter:title - 标题
+ twitter:description - 描述
+ twitter:image - 图片



### 3.5 Schema.org 结构化数据标签（推荐）
**用途说明**

帮助搜索引擎更好地理解页面内容，可能获得富文本摘要（Rich Snippets）展示。

**常用 itemprop 标签**

+ itemprop="name" - 内容名称
+ itemprop="description" - 内容描述
+ itemprop="image" - 内容图片

**更高级的结构化数据**

建议使用 JSON-LD 格式添加结构化数据（在 head 或 body 中添加），常见类型：

+ 文章类（Article）：标题、作者、发布时间、图片等
+ 组织类（Organization）：公司信息、Logo、联系方式等
+ 面包屑（BreadcrumbList）：导航路径
+ 评分类（AggregateRating）：用户评分信息
+ 问答类（FAQPage）：常见问题页面



### 3.6 其他重要 Meta 标签
**搜索引擎爬虫控制**

+ robots meta 标签：控制页面是否可被索引和抓取（如 index,follow）
+ canonical 标签：指定页面的规范版本，避免重复内容问题

**移动端优化**

+ format-detection meta 标签：控制手机号、邮箱等的自动识别

**浏览器兼容性**

+ X-UA-Compatible meta 标签：指定 IE 浏览器渲染模式

**国际化支持**

+ hreflang 标签：多语言网站指定不同语言版本（link 标签）



### 3.7 标签完整性检查清单
开发完成后，使用以下清单进行检查：

**基础必需项（100%页面必须有）**

+ ✓ Title 标签（唯一、符合长度要求）
+ ✓ Description meta 标签（唯一、符合长度要求）
+ ✓ Keywords meta 标签
+ ✓ UTF-8 编码声明
+ ✓ Viewport 设置

**社交分享优化（面向用户分享的页面必须有）**

+ ✓ OG:title
+ ✓ OG:description
+ ✓ OG:image（使用绝对路径）
+ ✓ OG:type
+ ✓ OG:url

**结构化数据（重要页面推荐有）**

+ ✓ Schema.org 标记（itemprop 或 JSON-LD）
+ ✓ 面包屑导航结构化数据
+ ✓ 文章/产品结构化数据

**爬虫控制（根据需要）**

+ ✓ Robots meta 标签（需要控制的页面）
+ ✓ Canonical 标签（有重复内容风险的页面）

## 四、页面结构化标签优化规范
### 4.1 标题标签（H1-H6）优化规范
**H1 标签规范**

+ 唯一性原则：每个页面有且仅有一个 H1 标签
+ 位置要求：H1 应该是页面主标题，通常在顶部显著位置
+ 内容要求：
    - 包含页面核心关键词
    - 与 Title 标签内容相关但不完全相同
    - 长度控制在 20-50 个字符
    - 语义清晰，描述页面主题
+ 常见错误：Logo 使用 H1、一个页面多个 H1、H1 内容与页面主题无关

**H2-H6 标签规范**

+ 层级结构：严格按照 H1 > H2 > H3 的层级关系使用，不可跳级
+ H2 标签：用于页面主要章节标题，一个页面可有多个
+ H3-H6 标签：用于子章节和次级内容分类
+ SEO 价值：H2-H3 对 SEO 仍有价值，H4-H6 价值较低
+ 关键词分布：H2-H3 标签中适当融入相关关键词，但避免堆砌

**最佳实践示例**

页面结构建议：

+ H1：页面主标题（1 个）
+ H2：主要内容板块标题（3-5 个）
+ H3：板块内的小节标题（若干个）
+ H4-H6：更细分的层级（按需使用）



### 4.2 链接标签（A）优化规范
**锚文本优化**

+ 描述性原则：锚文本应清晰描述目标页面内容
+ 避免模糊词（忽略）：禁止使用"点击这里"、"查看更多"、"详情"等无意义文本
+ 关键词融入：锚文本中自然包含目标页面的关键词
+ 长度适中：锚文本建议 2-8 个字，不宜过长

**链接属性设置**

+ title 属性：为链接添加 title 属性，提供额外信息（鼠标悬停时显示）
+ rel 属性：
    - 外部链接使用 rel="nofollow"（不传递权重的链接）
    - 广告/付费链接使用 rel="sponsored"
    - 用户生成内容中的链接使用 rel="ugc"
    - 同时打开新窗口时使用 rel="noopener noreferrer"（安全性）
+ target 属性：站外链接建议使用 target="_blank" 新窗口打开

**内链策略**

+ 相关性原则：链接到相关性高的内部页面
+ 合理密度：每 300-500 字添加 1-2 个内链
+ 避免过度：同一页面不要重复链接到同一目标页面
+ 重要页面优先：多链接到希望搜索引擎重点抓取的页面

**外链策略**

+ 权威性原则：优先链接到高权重、权威性强的外部网站
+ 相关性原则：链接到与内容相关的外部资源
+ nofollow 使用：对于不信任的外链、广告链接、用户生成内容中的链接，使用 nofollow
+ 定期检查：避免死链，定期检查外链的有效性



### 4.3 图片优化规范
**Alt 属性（必需）**

+ 功能价值：
    - 图片无法加载时显示的替代文本
    - 帮助搜索引擎理解图片内容（图片 SEO 的核心）
    - 提升网站可访问性（屏幕阅读器会读取 alt 文本）
+ 内容要求：
    - 准确描述图片内容，5-15 个字为宜
    - 自然融入相关关键词（避免堆砌）
    - 每张图片的 alt 应该独特
    - 装饰性图片可使用空 alt（alt=""），但不要省略 alt 属性
+ 常见错误：使用图片文件名作为 alt、所有图片用相同的 alt、完全省略 alt 属性

**Title 属性（推荐）**

+ 鼠标悬停时显示的提示文本
+ 可提供比 alt 更详细的描述信息
+ 不是 SEO 的核心因素，但可提升用户体验

**图片文件名优化**

+ 使用描述性文件名，而非随机字符
+ 使用英文和连字符，如 seo-workflow-guide.jpg
+ 避免使用中文、空格、特殊字符

**图片格式和大小**

+ 格式选择：
    - 照片：使用 JPEG（压缩后质量好）
    - 图标/Logo：使用 PNG（支持透明）或 SVG（矢量图）
    - 动画：使用 WebP 或现代格式
+ 尺寸优化：
    - 根据显示尺寸提供合适大小的图片
    - 使用响应式图片（srcset 属性）
    - 单张图片大小尽量控制在 200KB 以内
+ 懒加载：对非首屏图片使用懒加载（loading="lazy" 属性），提升首屏加载速度

**图片结构化数据**

+ 对于重要图片（如产品图、文章配图），可添加 ImageObject 结构化数据
+ 包含图片 URL、描述、作者、版权信息等



## 五、搜索引擎内容可见性优化
### 5.1 内容开放策略
**核心原则：免费内容对搜索引擎完全开放**

+ 价值内容开放：将对用户有价值的核心内容（如免费题目、免费教程、文章全文）完全开放给搜索引擎
+ 避免登录墙：不要要求搜索引擎爬虫登录才能访问内容
+ 避免付费墙：免费内容不要设置付费限制（付费内容可设置，但需要正确标记）

**实施策略**

+ 题库类产品：免费题目的题干、题解、答案完全展示给搜索引擎
+ 教程类产品：免费教程的完整内容开放给搜索引擎
+ 内容平台：文章、问答等 UGC 内容对搜索引擎可见
+ 会员内容：可在页面中标注"会员专享"，但仍在 HTML 中渲染内容（使用 CSS 遮罩），或使用结构化数据标记付费内容



### 5.2 JavaScript 渲染问题
**问题说明**

虽然 Google 已支持 JavaScript 渲染，但抓取成本更高，可能影响收录速度和质量。百度等国内搜索引擎对 JS 渲染的支持仍然有限。

**解决方案**

+ **服务端渲染（SSR）：使用 Next.js、Nuxt.js 等框架实现 SSR**
+ 静态生成（SSG）：对于内容固定的页面，生成静态 HTML
+ 预渲染：使用 Prerender.io 等工具为搜索引擎提供预渲染的 HTML
+ 渐进增强：确保 HTML 中包含主要内容，JavaScript 仅用于增强交互

**检查方法**

+ 禁用浏览器 JavaScript，查看页面内容是否正常显示
+ 使用 Google Search Console 的 URL 检查工具查看 Google 抓取的内容
+ 查看网页源代码（右键-查看源代码），检查核心内容是否在 HTML 中



### 5.3 爬虫友好性设置
**Robots.txt 文件【重要】**

+ 位置：网站根目录（如 [https://example.com/robots.txt](https://example.com/robots.txt) ）
+ 功能：告诉搜索引擎哪些页面可以抓取，哪些不可以
+ 注意事项：
    - 不要误拦截重要页面
    - 可拦截后台管理页面、用户隐私页面、重复内容页面
    - 允许抓取 CSS 和 JS 文件（不要拦截）
+ 定期检查：确保 robots.txt 设置正确，未误拦截

**Sitemap 网站地图【重要】**

+ XML Sitemap：
    - 列出网站所有重要页面的 URL
    - 包含最后更新时间、更新频率、优先级
    - 建议按内容类型分多个 sitemap（文章、产品、分类等）
    - 提交到 Google Search Console、百度站长平台等
+ HTML Sitemap：
    - 为用户提供的网站结构导航页面
    - 有助于用户和搜索引擎了解网站结构

爬虫抓取速度

+ 确保服务器稳定，响应速度快（建议 < 200ms）
+ 合理设置 CDN 加速
+ 避免对爬虫 IP 进行限流（可在日志中识别爬虫 User-Agent）



### 5.4 移动端优化
**响应式设计 vs 独立移动站**

+ **响应式设计（推荐）**：一套代码适配所有设备，URL 统一，利于 SEO
+ 独立移动站：需要设置 canonical 和 alternate 标签，指明 PC 和移动版本的对应关系

**移动友好性要求**

+ 文字大小适中，无需缩放即可阅读
+ 点击元素间距足够（至少 48x48 像素）
+ 避免使用 Flash 等移动端不支持的技术
+ 页面加载速度快（建议 < 3 秒）

**AMP（加速移动页面）**

+ Google 推出的移动端加速技术
+ 国内使用较少，国际化产品可考虑
+ 可在移动搜索结果中获得闪电标识



## 六、内容质量和用户体验优化
### 6.1 内容原创性和质量
**原创内容的重要性**

+ 搜索引擎优先展示原创、高质量内容
+ 避免大量采集或复制其他网站内容
+ 对于必须引用的内容，注明出处并添加 canonical 标签

**内容质量标准**

+ **深度**：内容详实，提供真实价值，不是浅尝辄止
+ **专业性**：展示专业知识和权威性（E-A-T 原则）
+ **时效性**：及时更新过时内容，保持信息准确
+ **可读性**：排版清晰、逻辑清楚、配图合理
+ **长度**：根据主题深度决定，一般不少于 500 字



### 6.2 避免 TDK 重复
**重复问题的危害**

+ 搜索引擎无法判断哪个页面更重要
+ 可能被判定为低质量内容
+ 降低整站的 SEO 表现

**检查和解决方法**

+ **定期检查**：使用 Screaming Frog、Sitebulb 等工具检查全站 TDK 重复情况
+ **URL 参数处理**：对于筛选、排序等 URL 参数生成的页面，使用 canonical 标签指向标准版本
+ **分页处理**：列表页的分页使用 rel="prev" 和 rel="next" 标签（或设置 canonical 到第一页）
+ **动态生成**：详情页的 TDK 应从内容中动态提取，确保每页唯一



### 6.3 用户体验信号
**影响 SEO 的用户行为指标**

+ **点击率（CTR）**：搜索结果的点击率高，说明标题描述吸引人
+ **停留时间**：用户在页面停留时间长，说明内容有价值
+ **跳出率**：跳出率低，说明用户找到了所需内容
+ **回访率**：用户多次访问，说明网站有价值

**优化策略**

+ 优化 TDK，提升搜索结果点击率
+ 提供高质量内容，延长用户停留时间
+ 改善页面加载速度，降低跳出率
+ 增加内链，引导用户浏览更多页面
+ 提供订阅、收藏等功能，提升回访率

## 八、常见问题和注意事项
### 8.1 TDK 常见错误
**错误示例和解决方案**

1. **标题太短或太长**
    - 错误：标题只有品牌名"编程导航"
    - 正确：编程导航 - 程序员的成长社区 | 优质编程学习资源大全
2. **描述中堆砌关键词**
    - 错误：Java、Java 教程、Java 面试、Java 学习、Java 资源、Java 刷题...
    - 正确：自然地将关键词融入通顺的句子中
3. **关键词设置过多**
    - 错误：20 个关键词罗列
    - 正确：3-5 个核心关键词
4. **TDK 与页面内容不符**
    - 错误：标题是"Java 教程"，页面内容是 Python 教程
    - 正确：确保 TDK 与页面实际内容高度相关

### 8.2 结构化标签常见错误
1. **多个 H1 标签**
    - 一个页面只能有一个 H1
2. **标题层级跳跃**
    - 不要从 H1 直接跳到 H3
3. **图片缺少 alt**
    - 所有图片都应该有 alt 属性
4. **锚文本不明确**
    - 避免使用"点击这里"、"更多"等模糊词汇

### 8.3 技术实施注意事项
**单页应用（SPA）的 SEO**

+ 使用服务端渲染（SSR）或预渲染
+ 确保搜索引擎能抓取到完整内容
+ 每个路由都有独立的 TDK

**多语言网站 SEO**

+ 使用 hreflang 标签指定语言版本
+ 每个语言版本都有独立的 TDK
+ URL 结构清晰（如 /en/, /zh/）

**国际化 SEO**

+ 根据目标国家/地区选择合适的域名（.com, .cn 等）
+ 设置 Google Search Console 的目标国家
+ 考虑服务器地理位置（使用 CDN）

## 十、总结
本 SEO 工作流指南涵盖了从 TDK 设置、Meta 标签配置、页面结构优化到技术实施的完整流程。开发团队在实施时应遵循以下核心原则：

**三大核心原则**

1. 用户第一：优先考虑用户体验，SEO 优化应服务于用户需求
2. 内容为王：高质量、原创、有价值的内容是 SEO 的基础
3. 技术支撑：正确的技术实施确保搜索引擎能够发现、抓取和理解网站内容

**实施优先级**

+ 第一优先级：TDK 设置、H1 标签、图片 alt、移动端适配
+ 第二优先级：OG 标签、结构化数据、页面速度优化
+ 第三优先级：高级优化（AMP、国际化、多语言等）

**持续优化**

SEO 不是一次性工作，需要持续监控、分析和优化。建议每月进行一次 SEO 审计，根据数据反馈调整策略。

通过遵循本指南，开发团队能够系统化地完成网站 SEO 优化，提升网站在搜索引擎中的可见性和排名，最终带来更多高质量的自然流量。
